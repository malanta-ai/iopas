How Proximity Is Calculated

Every indicator is traced back to the closest seed along the minimum-cost chain of pivots. Each hop contributes a base cost reflecting link confidence; positional and predecessor adjustments then shape the final weighted distance w.

1. Base hop costs (lower = stronger link):

Strong (1.0)WHOIS email match, dedicated reverse DNS, cert serial match, co-hosted on same IP, subdomain, exact repo match, cluster (domain / email)
Moderate (1.2–1.5)DNS A/AAAA (1.2), infra pivot (1.2), repo subdomain (1.3), CNAME (1.5), cert CN (1.5), redirect (1.5), hosted IP (1.5), subdomain co-hosted (1.5), repo subnet / co-occurrence (1.5)
Weak (2.0+)HTML reference (2.0), unknown hop (2.0), shared reverse DNS (2.5), cluster via IP (2.5)

2. Positional amplification — later hops in a chain cost more (the “telephone effect”). Base cost at hop position k (0-indexed, counted from the seed) is multiplied by:

Position01234+
Multiplier1.00×1.05×1.20×1.45×1.75×

3. Predecessor bump — if the preceding hop was weak-class (base cost ≥ 2.0, e.g. shared reverse DNS or HTML reference), the current hop’s cost is multiplied by 1.2×. This captures the intuition that a pivot chained off a noisy predecessor is itself less trustworthy.

4. Path selection & normalization — when multiple pivot chains lead to the same indicator, the chain with the lowest weighted cost wins. The raw w is then mapped into pivot distance D ∈ [0, 1]:

D = 1 − e−0.3 · max(0, w − 0.7)

The 0.7 floor means a single strong pivot (w ≈ 1.0) reads as “very close” rather than being penalized; the exponential beyond that keeps medium and long chains well-separated.
